---
title: "IRB vs Pry"
description: "A feature-by-feature comparison of IRB and Pry, and the areas where IRB is actively catching up."
---

IRB is on a mission to match feature parity with Pry. This page tracks the current state of that effort, highlights where IRB has advantages, and identifies areas where contributions are welcome.

<Note>
  IRB is Ruby's default REPL and ships with every Ruby installation. It is the default Rails console since Rails 7.1. No additional gems are required.
</Note>

## Feature comparison

| Feature | Pry | IRB | Notes |
|---------|-----|-----|-------|
| Supported Rubies | `>= 2.0` | `>= 2.7` | |
| Source code browsing | `show-source` | `show_source` | IRB's `show_source` cannot display C source — see [#664](https://github.com/ruby/irb/issues/664) |
| Document browsing | `ri` | `show_doc` | |
| Live help system | `help` or `command_name --help` | `help` | IRB does not yet support detailed per-command descriptions via `--help` |
| Open methods in editors | `edit` | `edit` | |
| Syntax highlighting | Yes | Yes | |
| Command shell integration | Yes | No | No current plan to support shell passthrough in IRB |
| Navigation | `cd object`, `cd ..`, `nesting` | `cd object`, `cd ..`, `pushws`, `popws`, `workspaces` | Added to IRB in [v1.14.0](https://github.com/ruby/irb/releases/tag/v1.14.0) |
| Runtime invocation | `binding.pry` | `binding.irb` | |
| Command system | Yes | No | Planned in [#513](https://github.com/ruby/irb/issues/513) and [#588](https://github.com/ruby/irb/issues/588) |
| Input history | Comprehensive support | `history` command with `-g` filtering | The `history` command does not yet support as many flags as Pry |
| Pager support | Command output and return value | Command output and return value | |
| Debugger integration | `pry-byebug` gem | Built-in `irb:rdbg` sessions via `debug.gem` | |
| Rails console | `pry-rails` gem | Rails' default | IRB does not have Rails-specific commands like `show-routes` or `show-models` |

## Where IRB leads

<AccordionGroup>
  <Accordion title="Built into Ruby — no installation required">
    IRB ships as a default gem with every Ruby installation from 2.7 onward. You do not need to add anything to your Gemfile to use it. Pry requires an explicit dependency.
  </Accordion>
  <Accordion title="Default Rails console">
    Since Rails 7.1, IRB is the default console. You get syntax highlighting, autocompletion, and all built-in commands out of the box when you run `rails console`.
  </Accordion>
  <Accordion title="Official extension API">
    IRB v1.13.0 introduced a stable public API for extending IRB with custom commands and helper methods. See the [Extending IRB](/extending/overview) section for details.
  </Accordion>
  <Accordion title="debug.gem integration">
    IRB has first-class integration with Ruby's official `debug` gem. After hitting a `binding.irb` breakpoint, run `debug` to activate a full `irb:rdbg` session with step-debugging, frame navigation, thread control, and more — while retaining access to all IRB commands like `show_source` and autocompletion.
  </Accordion>
  <Accordion title="Type-aware autocompletion">
    IRB offers an opt-in `IRB::TypeCompletor` that uses runtime type analysis to complete chained method calls and block parameters — capabilities that go beyond what Pry's completion offers.
  </Accordion>
</AccordionGroup>

## Where Pry still leads

<AccordionGroup>
  <Accordion title="Command shell integration">
    Pry allows you to run shell commands directly from the REPL. IRB has no equivalent feature and there is currently no plan to add one.
  </Accordion>
  <Accordion title="C source browsing">
    Pry's `show-source` can display the C source for methods implemented in Ruby's C extensions. IRB's `show_source` returns an error for those methods. Track progress on [#664](https://github.com/ruby/irb/issues/664).
  </Accordion>
  <Accordion title="Plugin/command system">
    Pry has an established ecosystem of plugins and a command system that lets plugins register new commands. IRB's official extension API is newer and still growing.
  </Accordion>
  <Accordion title="Rails-specific commands">
    The `pry-rails` gem adds commands like `show-routes` and `show-models` to the Rails console. IRB does not currently have equivalents.
  </Accordion>
  <Accordion title="Input history flags">
    Pry supports a wider range of flags for navigating input history. IRB's `history` command currently supports `-g`/`-G` filtering but lacks other options.
  </Accordion>
</AccordionGroup>
