Bug reports and pull requests are welcome on GitHub at [https://github.com/ruby/irb](https://github.com/ruby/irb).

## Development setup

<Steps>
  <Step title="Fork and clone the repository">
    Fork the project to your GitHub account, then clone your fork:

    ```shell
    git clone git@github.com:[your_username]/irb.git
    cd irb
    ```
  </Step>
  <Step title="Install dependencies">
    ```shell
    bundle install
    ```
  </Step>
  <Step title="Verify the test suite passes">
    ```shell
    bundle exec rake
    ```

    All tests should pass before you make any changes.
  </Step>
</Steps>

## Running tests

### Unit tests

The standard test suite runs with:

```shell
bundle exec rake
```

### Integration tests

If your changes affect component rendering — such as the autocompletion dialog or dropdown — you need to run IRB's integration tests, which use the `yamatanooroti` framework.

These tests require `libvterm`. Install it before running them:

<Tabs>
  <Tab title="macOS (Homebrew)">
    ```shell
    brew install libvterm
    ```
  </Tab>
  <Tab title="Other platforms">
    Consult your package manager for a `libvterm` package. On Debian/Ubuntu, try:

    ```shell
    sudo apt-get install libvterm-dev
    ```
  </Tab>
</Tabs>

After installing `libvterm`, run:

```shell
WITH_VTERM=1 bundle install
WITH_VTERM=1 bundle exec rake test test_yamatanooroti
```

## Updating documentation

IRB's documentation is generated by [RDoc](https://ruby.github.io/rdoc/) and published to [ruby.github.io/irb](https://ruby.github.io/irb). Most source lives under the `doc/` directory.

To generate the documentation site locally:

```shell
bundle exec rake rdoc
```

To force a full regeneration:

```shell
bundle exec rake rerdoc
```

Follow the output message to open the generated site in your browser.

<Warning>
  RDoc's Markdown support is limited. The rendered result on GitHub may differ from what appears on [https://ruby.github.io/irb](https://ruby.github.io/irb). Always verify your changes render correctly by generating the docs locally before opening a pull request.
</Warning>

Documentation contributions are welcome, including:

- Fixing typos and grammatical errors
- Adding missing documentation for features or configuration options
- Adding demo images or GIFs for features

## Submitting issues

Before opening an issue:

- Check the [existing issues](https://github.com/ruby/irb/issues) to see if it has already been reported.
- Include your Ruby version (`ruby --version`) and IRB version (`irb --version`) in the report.
- If you can reproduce the problem, include a minimal script or IRB session that demonstrates it.

## Opening a pull request

<Steps>
  <Step title="Create a branch">
    ```shell
    git checkout -b my-feature
    ```
  </Step>
  <Step title="Make your changes">
    Add your code and tests. IRB uses the standard Ruby test framework. Look at existing test files under `test/` for examples.
  </Step>
  <Step title="Run the test suite">
    ```shell
    bundle exec rake
    ```

    Ensure all tests pass before submitting.
  </Step>
  <Step title="Push and open a PR">
    ```shell
    git push origin my-feature
    ```

    Open a pull request against the `main` branch of [ruby/irb](https://github.com/ruby/irb) on GitHub. Describe what your change does and why.
  </Step>
</Steps>

<Tip>
  For larger changes, consider opening an issue first to discuss the approach before investing time in an implementation.
</Tip>
