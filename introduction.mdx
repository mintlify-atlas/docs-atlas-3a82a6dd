IRB stands for "Interactive Ruby." It is a command-line tool that reads Ruby expressions, evaluates them, and prints the result — immediately, as you type. It ships with every Ruby installation, so there is nothing to set up.

## What IRB is

IRB operates as a **Read-Eval-Print Loop** (REPL):

- **Reads** each character as you type, with syntax-aware multi-line editing
- **Evaluates** the code once it forms a syntactically complete expression
- **Prints** the result using `inspect`, with syntax highlighting

Input is buffered until your code is syntactically complete. You can define a class across several lines and IRB will not evaluate it until the closing `end`:

```shell
$ irb
irb(main):001> 1 + 2
=> 3
irb(main):002* class Foo
irb(main):003*   def foo
irb(main):004*     puts 1
irb(main):005*   end
irb(main):006> end
=> :foo
irb(main):007> Foo.new.foo
1
=> nil
```

The `>` prompt means IRB is ready for a new expression. The `*` prompt means the current expression is incomplete and IRB is waiting for more input.

## Who uses IRB

- **Ruby developers** experimenting with language features or prototyping small scripts
- **Rails developers** using `rails console`, which runs on IRB
- **Anyone debugging** Ruby code using `binding.irb` breakpoints

## How it fits into the Ruby ecosystem

IRB is a **default gem** — it is bundled with Ruby and available immediately after installing Ruby. You do not need to install anything extra to start using it.

`rails console` in a Rails application opens an IRB session with the Rails environment loaded. Any IRB configuration, commands, or extensions you set up will work there too.
