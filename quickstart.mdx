title: "Quickstart"
description: "Start an IRB session, evaluate Ruby expressions, define classes, use binding.irb as a breakpoint, and run built-in commands."
---

This guide walks you through the core IRB workflow from starting a session to using built-in commands and breakpoints.

<Steps>
  <Step title="Start IRB">
    Open a terminal and run:

    ```shell
    $ irb
    irb(main):001>
    ```

    IRB is included with Ruby — no installation required. The prompt `irb(main):001>` tells you IRB is ready for input. The `main` refers to Ruby's top-level object, and `001` is the line number.

    To exit at any time, type `exit` and press Enter, or press `Ctrl+D`.
  </Step>

  <Step title="Evaluate basic expressions">
    Type any Ruby expression and press Enter. IRB evaluates it and prints the return value after `=>`:

    ```shell
    irb(main):001> 1 + 2
    => 3
    irb(main):002> "hello".upcase
    => "HELLO"
    irb(main):003> [1, 2, 3].map { |n| n * 2 }
    => [2, 4, 6]
    ```

    The `=>` prefix marks the return value of the evaluated expression.
  </Step>

  <Step title="Write multi-line input">
    IRB detects when an expression is syntactically incomplete and waits for more input. The prompt changes from `>` to `*` while input is open:

    ```shell
    irb(main):001> class Foo
    irb(main):002*   def foo
    irb(main):003*     puts 1
    irb(main):004*   end
    irb(main):005> end
    => :foo
    irb(main):006> Foo.new.foo
    1
    => nil
    ```

    IRB evaluates the class definition only after the closing `end` makes it syntactically complete. Automatic indentation helps you track the nesting level.
  </Step>

  <Step title="Use binding.irb as a breakpoint">
    You can pause a running Ruby script and open an IRB session in its exact context using `binding.irb`. Create a file `test.rb`:

    ```ruby
    def greet(word)
      binding.irb
      puts "Hello #{word}"
    end

    greet("World")
    ```

    Run the file:

    ```shell
    $ ruby test.rb

    From: test.rb @ line 2 :

        1: def greet(word)
     => 2:   binding.irb
        3:   puts "Hello #{word}"
        4: end
        5:
        6: greet("World")

    irb(main):001> word
    => "World"
    irb(main):002> exit
    Hello World
    ```

    IRB opens with the local variable `word` in scope. You can inspect variables, call methods, and evaluate expressions in the live context of your program. Type `exit` to resume execution.
  </Step>

  <Step title="Explore objects with built-in commands">
    IRB includes commands for inspecting objects and source code. These are not Ruby methods — they are IRB commands available at the prompt.

    **`ls` — list methods and constants**

    ```shell
    irb(main):001> ls String
    String.methods: try_convert
    String#methods:
      %  *  +  <<  <=>  ==  ===  =~  []  []=  ascii_only?  b  bytes  bytesize
      byteslice  capitalize  capitalize!  casecmp  casecmp?  center  chars  chomp
      ...
    ```

    **`show_source` — display source code**

    ```shell
    irb(main):001> show_source String#upcase
    ```

    **`show_doc` — look up documentation**

    ```shell
    irb(main):001> show_doc Array#flatten
    ```

    **`help` — list all available commands**

    ```shell
    irb(main):001> help
    ```

    <Tip>
      `$` is an alias for `show_source` and `@` is an alias for `whereami`. You can check your current aliases with `conf.command_aliases`.
    </Tip>
  </Step>
</Steps>

## Exit IRB

| Method | Effect |
|--------|--------|
| `exit` | Exit the current IRB session |
| `exit!` | Exit the current process immediately |
| `Ctrl+D` | Send end-of-file to exit (same as `exit` by default) |
